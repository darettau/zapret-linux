# tg-proxy

Локальный MTProto-прокси для клиента Telegram. Telegram подключается к
`127.0.0.1:1443`, прокси заворачивает трафик в TLS-соединения к дата-центрам
Telegram — для DPI это выглядит как обычный HTTPS, поэтому блок MTProto обходится.

## Запуск

Проще всего — стратегией 6 из корня проекта:

```bash
sudo ./start.sh 6
```

Либо напрямую (root не нужен):

```bash
./run.sh        # слушает 127.0.0.1:1443
```

Затем в Telegram: **Настройки → Продвинутые → Тип соединения → Свой прокси →
MTProto**, сервер `127.0.0.1`, порт `1443`, secret — из вывода скрипта (строка
`dd…`). Либо открыть напечатанную ссылку `tg://proxy?...`.

Без VPN прокси ходит к доменам Telegram по TLS (443), поэтому полезно поднять
шифрованный DNS (`../setup-doh.sh`) и обход DPI (`../start.sh 8`).

Внешних Python-зависимостей нет: стандартная библиотека + системный OpenSSL
через ctypes (или пакет `cryptography`, если установлен).
